Layer 3 Cloud Functions — バックグラウンド処理 定時チェック · メール送信 · 通知 ▶
毎日の処理フロー（JST 09:00実行）
09:00 JST — dailyCheckJob
全ユーザーの users コレクションを走査。paused=true のユーザーはスキップ。
各ユーザー — 状態判定
checkins/{today} が存在 → safe (スキップ)
checkins/{today} なし, checkins/{yesterday} あり → pending (スキップ)
checkins/{yesterday} なし, checkins/{2-days-ago} あり → warn → FCM push送信
上記以外 → alert → メール送信（1回のみ）
warn 状態 — FCM Push 送信
users/{uid}.fcmToken へ Firebase Admin でプッシュ送信
タイトル: 「昨日から確認がありません」
本文: 「明日までに確認しないと緊急連絡先へメールが届きます」
→ users/{uid}.fcmToken 使用 · 無料
alert 状態 — Resend メール送信
条件: users/{uid}.lastNotifiedAt が今日でない（重複送信防止）
contact.email へ Resend API で送信
件名: 「○○さんの様子をご確認ください」
送信後: lastNotifiedAt = now, emailSentCount++ を更新
→ Resend API · 3,000封/月無料

Cloud Function コードイメージ
// functions/src/dailyCheck.ts exports.dailyCheckJob = functions .region('asia-northeast1') // Tokyo .pubsub.schedule('every day 00:00') .timeZone('Asia/Tokyo') .onRun(async (context) => { const users = await db.collection('users').get(); const today = getJSTDateString(); // "2025-11-15" const yesterday = getJSTDateString(-1); const twoDaysAgo = getJSTDateString(-2); await Promise.all(users.docs.map(async (userDoc) => { const { paused, fcmToken, lastNotifiedAt } = userDoc.data(); if (paused) return; // 機能停止中はスキップ const todayCI = await checkInExists(userDoc.id, today); if (todayCI) return; // 確認済み const yesterdayCI = await checkInExists(userDoc.id, yesterday); const twoDaysCI = await checkInExists(userDoc.id, twoDaysAgo); if (!yesterdayCI && twoDaysCI) { await sendPushNotification(fcmToken); // warn } if (!yesterdayCI && !twoDaysCI) { if (!isSameDay(lastNotifiedAt, today)) { // 重複防止 await sendEmergencyEmail(userDoc.id); // alert } } })); });
Layer 4 Resend — メール設計 2種類のメール · 日本語テンプレート ▶
📨
① 確認メール（連絡先登録時）
To: 緊急連絡先のメール
件名: 「あなたは○○さんの緊急連絡先に登録されました」
本文: アプリの説明、同意確認リンク
目的: 連絡先の合意取得・スパム防止
送信タイミング: 登録時1回
🆘
② 緊急通知メール（3日未確認時）
To: 緊急連絡先のメール
件名: 「○○さんの様子をご確認ください」
本文: 「3日間応答がありません。手違いの可能性もあります。念のためご確認ください。」
重要: 「死」「緊急」の文言を使わない
送信タイミング: 1ユーザー1回のみ

Layer 6 環境構築チェックリスト Firebase設定 → Resend → Codemagic ▶
① Firebase セットアップ
□ Firebase Console でプロジェクト作成
□ iOS App 登録（Bundle ID: jp.kyogen.app など）
□ GoogleService-Info.plist をダウンロード → Flutter プロジェクトに配置
□ Blaze プランへアップグレード（クレカ登録のみ、課金は発生しない）
□ Firestore データベース作成（asia-northeast1 / Tokyo）
□ Authentication → Google Sign-In 有効化
□ Cloud Functions デプロイ（firebase deploy --only functions）
② Resend セットアップ
□ resend.com でアカウント作成
□ ドメイン設定（独自ドメインがあれば理想、なくても noreply@resend.dev で可）
□ API Key 発行
□ Firebase Functions の環境変数に設定: firebase functions:config:set resend.key="re_xxx"
③ APNs（iOS Push通知）
□ Apple Developer Console → Certificates → APNs キー作成
□ .p8 ファイルを Firebase Console → Project Settings → Cloud Messaging にアップロード
□ FCM は自動で APNs をブリッジ → コード側は FCM だけ書けばOK
④ Codemagic（クラウドビルド）
□ codemagic.io アカウント作成
□ GitHub リポジトリ連携
□ Apple Developer 証明書・Provisioning Profile を Codemagic に登録
□ GoogleService-Info.plist を Codemagic の Environment Variables に設定（セキュリティ）
□ codemagic.yaml をリポジトリのルートに配置
□ push → 自動ビルド → .ipa 生成 → TestFlight 自動アップロード
